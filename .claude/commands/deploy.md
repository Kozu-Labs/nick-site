Deploy nick-site to Firebase production. Execute these steps in order. Stop and report if any step fails — do not continue.

## Steps

1. **Pull latest with rebase**: `git pull --rebase origin main`. If conflicts, stop and report — do NOT auto-resolve.

2. **Persist memory + todos to `~/NS.md`** (the unified firm-wide memory file; symlink to iCloud workspace):
   a. Add a session-summary entry to § 4 "Session log" (newest first) describing what was built/changed/decided this session. Look at the conversation context for the work — this should capture the meaningful narrative, not just a deploy record.
   b. Update § 2 "Active todos" to reflect what was completed (`[x]`) and what's still open (`[ ]`). Pull the current TodoWrite state and reconcile with the file.
   c. If new conventions or hard rules were established, add them to § 3.
   d. Mirror to the repo: `cp ~/NS.md "$(git rev-parse --show-toplevel)/MEMORY/NS.md"`.

3. **Stage source files**: `git add` everything EXCEPT build output. For nick-site:
   - INCLUDE: `public/index.html`, `public/portals/**`, `public/css/`, `public/images/`, `firestore.rules`, `storage.rules`, `firebase.json`, `functions/*.js`, `functions/package.json`, `functions/package-lock.json`, `MEMORY/NS.md`, etc.
   - EXCLUDE: `public/blog/` (regenerated by `build-blog.py` from `blog-content/`), `functions/node_modules/`, `.firebase/`, `.DS_Store`.

4. **Commit + push**: short descriptive commit message ending with `Co-Authored-By: Claude Opus 4.7 (1M context) <noreply@anthropic.com>`. Push to `origin main`. **Push must succeed before deploy.**

5. **Deploy**:
   - Always: `firebase deploy --only hosting`
   - If functions changed: `firebase deploy --only hosting,functions`
   - If firestore.rules or storage.rules changed: include `firestore` / `storage` in the --only list

6. **Verify**: `curl -I https://nick-site-web.web.app/` returns 200. Report Hosting URL + commit SHA in chat.

## Hard rules

- Never deploy before pushing.
- Never commit `public/blog/` (regenerated from `blog-content/` by `build-blog.py`).
- Never commit `functions/node_modules/` or `.firebase/`.
- Always rebase, never merge.
- If push fails (remote has changes), pull-rebase again before retrying.
- If deploy fails, the source code IS pushed — that's intentional. Investigate and fix forward; don't roll back the commit.
- `~/NS.md` is a symlink to `Nick Lawfirm AI/NS.md` (iCloud workspace). Edits via `~/NS.md` are fine; they go to the iCloud file.
